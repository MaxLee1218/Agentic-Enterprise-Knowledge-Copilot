# Supplier Quality Analysis 业务范围基线

## 1. 场景描述

首版仅支持 **Supplier Quality Analysis（供应商质量分析）**。典型请求为：

> Analyze supplier quality issues in Q1 and generate a quality analysis report.

系统在已授权的企业知识库和质量数据库内，将用户请求转换为受约束任务，检索适用的质量政策与供应商资料，执行只读查询，完成确定性指标计算，生成并验证分析报告，同时保留从请求到报告结论的证据链。

本版本是有限业务域内的可靠执行器，不是通用自主 Agent。所有数据范围、工具、状态转换、审批和失败处理均由显式契约约束。

## 2. 用户目标

质量管理人员需要在指定季度和供应商范围内：

1. 识别缺陷率、缺陷数量和趋势等质量表现；
2. 结合企业质量政策和供应商资料解释发现；
3. 区分事实、确定性计算结果和分析性判断；
4. 获得一份可供管理层审阅的质量分析报告；
5. 从每项重要结论追溯到查询、计算或文档证据。

## 3. 输入

| 输入 | 必需 | 约束 |
|---|---:|---|
| 原始业务请求 | 是 | 自然语言；必须能够归类为供应商质量分析 |
| 报告期间 | 是 | 明确年份、季度及其闭区间日期；“Q1”不能脱离年份执行 |
| 供应商范围 | 否 | 未指定时使用用户获准访问的全部供应商；实际范围写入任务契约 |
| 质量指标 | 否 | 默认包含缺陷数、检验批次/数量、缺陷率和期内趋势 |
| 输出格式 | 否 | 首版支持结构化报告和 PDF 报告；默认 PDF |
| 用户与租户上下文 | 是 | 来自可信认证上下文，不从自然语言自行推断 |
| 数据与文档访问范围 | 是 | 由权限和数据访问策略计算，并固定在任务契约中 |
| 审批信息 | 条件必需 | 当策略判定数据范围、报告内容或操作需要审批时提供 |

若年份、时间范围或执行所需的授权上下文缺失，任务不能进入执行；系统以可恢复错误返回补充要求。

## 4. 输出

1. `TaskResult`：任务最终状态、简要结论、Artifact 与 Evidence 引用集合。
2. 质量分析报告：至少包含范围与口径、指标结果、趋势、主要发现、限制、证据引用和生成时间。
3. 证据清单：文档证据、数据库查询证据和计算证据；每项均具有稳定标识和来源引用。
4. 审计轨迹：记录状态变化、计划版本、工具调用、策略决定、审批决定、重试、重规划和验证结果。
5. 若失败或取消：返回类型化 `TaskError`，不生成“成功”报告；可保留已完成步骤的合规证据用于诊断和恢复。

## 5. In Scope

- 在批准范围内检索企业知识库；
- 对注册的质量数据库执行参数化、单语句、只读 `SELECT` 查询；
- 使用确定性算法计算缺陷率、计数、分组汇总和时间趋势；
- 基于已验证分析结果与证据生成报告；
- 在策略要求时创建、等待和处理人工审批；
- 验证证据覆盖、数字一致性、范围一致性和 Artifact 完整性；
- 对瞬时技术失败执行有界重试，对可修复的计划或验证问题执行有界重规划；
- 支持用户取消、检查点恢复、审计记录和端到端追踪。

## 6. Success Criteria

任务仅在以下条件全部满足时标记为 `COMPLETED`：

1. 请求被明确归类为 `supplier_quality_analysis.v1`，年份、季度、租户和数据范围已固定；
2. 执行计划通过结构、依赖、权限和能力校验；
3. 所有必需步骤成功，或产生契约明确允许的业务结果（例如数据库空结果）；
4. 所有数据库访问均为获准范围内的只读查询，且记录查询指纹、参数摘要和结果行数；
5. 指标由确定性分析工具计算，单位、分母、时间窗口及算法版本可追溯；
6. 报告中的每项重要事实和数字结论至少关联一个适配类型的证据项；
7. 报告数字与计算证据一致，引用可解析，Artifact 可读取且校验和匹配；
8. 所有必需审批均为有效的 `APPROVED`，且审批范围覆盖实际执行范围；
9. 无未解决的高严重度验证问题；
10. 从 `TaskRequest → TaskContract → TaskPlan/TaskStep → ToolCall/ToolResult → EvidenceItem → TaskResult → Artifact` 可完整追溯。

业务空结果不等于失败。若获准范围内确实没有缺陷记录，系统生成“该范围内无记录”的报告，明确数据覆盖和限制，并仍需通过验证。

## 7. Out of Scope

- 自动执行 `INSERT`、`UPDATE`、`DELETE`、`ALTER`、`DROP` 或任何数据库写入；
- 自动发送邮件、消息、报告或进行其他外部发布；
- 自动创建采购订单、付款、索赔、处罚或供应商整改动作；
- 自动修改供应商等级、状态、准入或黑名单；
- 对供应商质量根因作无证据的确定性归因或作法律/财务决策；
- 多 Agent 协作、Agent 间委派或自治群组；
- 跨天运行、无限观察、后台长期自主任务；
- 实时流处理、预测性质量模型和非确定性统计建模；
- 任意 SQL、任意知识源、开放互联网搜索或用户临时接入的数据源；
- MCP 客户端或服务端互操作；该能力属于后续阶段；
- 自动绕过审批、扩大权限或将模型文本视为授权；
- 除供应商质量分析外的采购、财务、人力资源等业务场景。

超出范围的请求在理解阶段以 `UNSUPPORTED_TASK_TYPE` 或 `SCOPE_VIOLATION` 明确拒绝，不会通过提示词或临时工具组合隐式支持。
